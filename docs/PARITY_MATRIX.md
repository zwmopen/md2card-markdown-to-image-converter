# MD2Card 当前版本复刻差距矩阵

> 目标顺序：先学习当前产品，再实现功能与视觉等价，最后发展为支持批量任务、私有部署和自有 MCP 的独立产品。
>
> 调研日期：2026-07-10。以下矩阵以当前中国站、旧版站和官方文档为参照，不再把仓库中早期静态页面误认为完整产品。

## 五、当前已完成的工程底盘

- 已将占位脚本替换为真实运行时。
- 已实现 Markdown 预览、自动分页、文件/文件夹导入、样式控制、PNG 与 ZIP 导出。
- 已使用经典脚本链保留直接打开本地 HTML 的兼容性。
- 已增加语法检查、核心算法测试、界面契约测试、本地加载兼容测试和隐藏分页导出测试。
- 第一阶段功能基线已通过 GitHub Actions 并合并至 `master`。

## 六、下一轮验收顺序

1. 固化当前官网页面、主题和关键交互的截图样本。
2. 建立官方主题 ID、模式和尺寸参数的兼容层。
3. 把字符估算分页升级为基于真实 DOM 高度的确定性分页。
4. 补齐表格、数学公式、Mermaid、MDX、图片尺寸和分栏。
5. 增加 Playwright 浏览器端功能测试与视觉回归。
6. 完成等价 Web 版本后，再抽离 Render Core、API、CLI 和批量 MCP。

## 七、边界

“1:1”指功能、信息架构、主要交互、参数语义和视觉结果达到可验证等价；不直接复制第三方密钥、用户数据、受限素材或无法确认授权的商业字体。自有产品会保留独立品牌和可替换主题资产。
