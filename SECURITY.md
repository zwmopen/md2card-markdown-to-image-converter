# Security Policy

## 数据与隐私边界

- 不提交 API Key、访问令牌、Cookie、账号密码、客户资料或个人隐私。
- Markdown 输入、背景图、品牌模板和导出结果均应视为用户数据，不应默认上传到第三方服务。
- 浏览器端导入的文件默认只在本地内存中处理；新增远程 API、统计或云端任务前，必须明确说明数据去向。
- 第三方 CDN、字体、脚本和主题资产必须记录来源、用途和替换方案。
- 不复制或分发无法确认授权的商业字体、贴纸、模板和品牌素材。

## 渲染安全

- Markdown、HTML、MDX、Mermaid 和外部图片地址都属于不可信输入。
- 新增 HTML 渲染能力时必须防止任意脚本执行、危险 URL、事件属性和数据泄露。
- 文件名、ZIP 路径和批量任务目录必须清理路径穿越字符。
- 导出失败不得生成空白文件并伪装成成功结果。

## 报告问题

请不要在公开 Issue 中粘贴密钥、私有文档或客户内容。优先使用 GitHub Security 页面或私下联系维护者，并提供最小复现、受影响版本和数据风险范围。
